# AtomSSH 远端到远端文件传递

>

## 1. 场景

运维经常需要在云 VPC 内两台机器之间传递文件。两台机器可能都不能直接暴露公网，只能通过跳板机、代理链或当前运维电脑中转。AtomSSH 应提供明确的“远端到远端复制”工作流，让用户通过选择源机器、目标机器和路径，尽量用两三次点击完成常见文件传递。

## 2. 传递模式

第一版只实现一种模式：

- LocalRelay：源机器 SFTP 下载到本机临时流，再上传到目标机器。

LocalRelay 更通用，不要求源机器和目标机器彼此可达，但流量经过本机。RemoteCommand、scp 和 rsync 编排属于后续增强，不进入第一版。

## 3. UI 职责

页面需要用户选择：

- 源 profile 和源路径。
- 目标 profile 和目标路径。
- 传递模式。
- 覆盖策略。
- 是否保留临时文件。

页面只能创建传输任务，不直接持有两个 SFTP client。
