## 1. 布局原则

AtomSSH 第一版采用 SSH 工作台式主界面：

- 顶部为 TopBar。
- 左侧为会话和功能导航。
- 中间为终端标签工作区。
- 右侧第一版可显示 SFTP 面板或保持关闭。
- 底部为 StatusBar。

第一版不做复杂 Docking、不做多窗口布局、不做 Dashboard。

## 3. 区域职责

TopBar 只承载品牌和全局入口，不堆叠页面级操作。

Navigation 负责切换页面、展示 profile 分组和工具入口，不直接执行底层 SSH、SFTP、Transfer 或 Network 操作。

Terminal Workspace 是核心工作区，承载终端标签、连接状态、断开、重连和关闭。

SFTP Panel 用于当前活动会话的远程文件浏览和传输入口。第一版可以作为独立页面或右侧面板实现，但不能让 SFTP UI 直接持有 SFTP client。

StatusBar 展示当前连接、转发、传输和错误摘要。
